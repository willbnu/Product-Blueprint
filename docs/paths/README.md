# Documentation Journey Paths

**Welcome!** Choose the path that matches your project type for a guided journey through the documentation.

## 🗺️ Available Paths

### 📱 [Mobile-First Application](./mobile-first-app.md)
**Best for:** iOS and Android apps (React Native/Expo)

**You'll learn:**
- Expo Router setup and navigation
- Mobile-specific patterns and best practices
- NativeWind styling
- Mobile deployment with EAS

**Time commitment:** 3-4 weeks to production-ready MVP

---

### 🌐 [Web-First Application](./web-first-app.md)
**Best for:** Web applications only (React/Vite)

**You'll learn:**
- React Router v6 setup
- Vite build optimization
- Tailwind CSS patterns
- Web deployment (Vercel/Netlify)

**Time commitment:** 2-3 weeks to production-ready MVP

---

### 🚀 [Full-Stack Application](./full-stack-app.md)
**Best for:** Both mobile and web platforms (complete monorepo)

**You'll learn:**
- Nx monorepo architecture
- Shared library patterns
- Cross-platform UI components
- Multi-platform deployment

**Time commitment:** 4-6 weeks to production-ready MVP

---

### 🏥 [Compliance-Heavy Application](./compliance-heavy-app.md)
**Best for:** Healthcare, finance, or other regulated industries (HIPAA, GDPR, SOC2)

**You'll learn:**
- Compliance requirements and checklists
- Audit logging implementation
- Data encryption patterns
- Security-first architecture

**Time commitment:** 5-8 weeks to compliant MVP

---

### ⚡ [Quick MVP](./quick-mvp.md)
**Best for:** Rapid prototyping and validation

**You'll learn:**
- Minimum viable features
- Fastest path to deployment
- Core patterns only
- Quick validation strategies

**Time commitment:** 1-2 weeks to working prototype

---

## 🧭 How to Use These Paths

1. **Choose your path** based on your project type
2. **Follow the weekly breakdown** - each path is organized by week
3. **Read recommended docs** in the suggested order
4. **Focus on highlighted sections** - skip what's not relevant
5. **Use quick references** - bookmark key sections
6. **Build as you learn** - implement alongside reading

## 💡 Can I Switch Paths?

Yes! Paths are guides, not rules. You can:
- Start with Quick MVP, then follow Full-Stack path
- Combine Mobile + Compliance paths
- Cherry-pick sections from different paths

## 📚 Path Structure

Each path includes:

- **Weekly breakdown** - What to focus on each week
- **Recommended reading order** - Docs to read and in what sequence
- **Section highlights** - Specific sections to focus on
- **What to skip** - Save time by skipping irrelevant parts
- **Quick reference card** - Bookmark-worthy sections
- **Checkpoints** - Validate your progress
